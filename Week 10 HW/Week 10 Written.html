1. Explain what a REST API is. What makes an API "RESTful"?
    -A REST (REpresentational State Transfer) API is an API that conforms to the REST architecture. This means that the API uses HTTP requests (get to retrieve, post to add, put/patch to update, and deletet to delete).
  2. What is Node.js and why is it often used when building server-side applications?
    -Node.js is a runtime environment that is used for executing JS outside the browser. It is often used in builiding server-side applications since its main feature is that is lightweight, using only low-level "core" modules. Node.js can be expanded via downloading open source packages. As a result of this Node.js has very good perforamance, being able to handle more traffic with less hardware. It has also been widely adopted by large businesses (better performance = less cost).
  3. Express.js is a popular framework used in conjunction with Node.js. What are some of the key features of Express.js that make it useful for building web servers?
    -Like node, express is also minimalistic and lightweight. The three main features of express are the ability to define routes, process HTTP requests using middleware. and the ability to use a view engine to render dynamic templates
  4. In Express.js, what is middleware and how is it used?
    -Middleware are functions that run after the server recevies a request, but before a response is sent. Middleware has access to the request object, responses object, and is able to process the request before a response is sent.
  5. In terms of HTTP verbs, what is the difference between GET, POST, PUT, PATCH, and DELETE? When would you use each in the context of a RESTful API?
    -Get retrieves data, post adds data, patch updates data, and delete deletes data. In terms of a RESTful api, get is used to retrieve a list. add adds to that list, put updates a specific item from the list, and delete deletes an item fromt the list.
  6. How would you handle parameters in the URL with Express.js? Provide an example.
    -You would add your paramters after the route with /route/:parameter. You would then include the parameter in your response.
